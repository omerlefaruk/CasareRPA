# Review Plan Checklist

Use this checklist before any implementation begins.

## Confirm Plan Quality
- Clear goal and scope
- Defined success criteria
- Risks and alternatives documented
- Files and tests listed

## Rule Alignment
- Re-read relevant `../rules/` and `.brain/` docs
- Confirm architecture and layer boundaries
- Confirm node/UI conventions if applicable

## Approval
- Ask user to approve the plan before EXECUTE
